# 高考志愿填报辅助技能

本技能提供一套**分数段无关、科目类别无关**的高考志愿填报方法论，核心是用位次替代分数做比较，通过官方文件核实专业信息，设计科学的冲/稳/保梯度。若用户提供官方 PDF，可使用内置脚本做定向文本提取/OCR，辅助核实院校代码、专业组代码和组内专业。

**重要提示：本技能输出仅供志愿填报参考，不构成最终填报依据。最权威数据必须以各省教育考试院、官方志愿填报系统、阳光高考平台、各高校招生章程、《招生考试之友》《招生专业目录》等当年官方渠道和官方材料为准。最终提交前必须回到官方渠道逐项核对。**

---

## 快速开始：收集必要信息

开始之前，必须确认以下信息：

```
1. 省份
2. 科目类别（物理类/历史类，或旧高考文/理科）
3. 分数
4. 省内同科目类别位次（最关键）
5. 报考批次（普通本科批/提前批/专项等）
6. 志愿填报模式（见Step 0分流）
7. 志愿数量上限
8. 目标方向（专业/行业）
9. 城市偏好
10. 深造意向（读研/读博/直接就业）
11. 特殊限制（不接受的城市/专业类型/中外合作等）
```

信息不全时优先询问位次和志愿填报模式，其他可在过程中补充。

若省份或志愿模式不明确，读取 `references/province-config.md`，先输出规则配置摘要；配置未核实时，不要直接进入冲稳保结论。

## 使用强度分层

根据用户材料选择合适强度，不要一开始就全量 OCR：

| 模式 | 适用情况 | 做什么 |
|------|---------|-------|
| 轻量策略 | 用户只有省份、科类、位次、方向 | 先做规则分流、梯度框架、风险提示 |
| 志愿表核验 | 用户已有候选志愿表 | 查梯度、排序、调剂风险，列出必须核实字段 |
| 官方 PDF 核验 | 用户提供《招生专业目录》/《招生考试之友》PDF | 用 `scripts/extract_admission_pdf.py` 定向提取候选院校/组号片段；需要表格时用 `--mode rows --format csv` |
| 最终复核 | 临近提交 | 回到官方 PDF 原页或考试院系统逐项核对代码、组号、专业、服从调剂 |

OCR 是最后核验工具，不是全流程前置条件。只有当用户提供官方 PDF、截图或明确要求提取目录时，才加载 `references/ocr-workflow.md` 并运行脚本。

## Step 0：规则分流（必须先做）

**不同省份、不同批次的填报规则差异极大，必须先确认模式再走对应流程。**

### 分流判断

```
问考生：你所在省份今年普通本科批是什么志愿模式？
```

可用 `references/province-config.md` 作为配置起点，但必须以当年省教育考试院公告为准。

| 志愿模式 | 代表省份 | 走哪个流程 |
|---------|---------|----------|
| 院校专业组模式 | 河南、湖南、湖北、广东、江苏、重庆、辽宁等 | → 走本文主流程（Step 1-7） |
| 专业+院校模式 | 北京、上海、天津、山东、海南等 | → 走专业志愿流程（见下方） |
| 传统院校志愿/顺序志愿 | 部分尚未完成新高考改革的省份 | → 走传统志愿流程（见下方） |

### 特殊批次提醒

以下批次**不走普通批模板**，需单独处理：

```
⚠️ 提前批（军校/公安/定向/师范等）：有额外资格要求，单独填报
⚠️ 国家专项/地方专项/高校专项：需要资格认定，不能混入普通批判断
⚠️ 强基计划：先通过校测，录取逻辑和普通批完全不同
⚠️ 综合评价招生：高考成绩+校测成绩，非普通批投档逻辑
⚠️ 艺术/体育类：综合成绩投档，另有专门规则
```

如果考生涉及上述特殊批次，先单独处理特殊批次，再处理普通批。

### 专业+院校模式流程要点

适用省份：北京、上海、天津、山东、海南等

**与院校专业组模式的核心差异：**
- 每个志愿直接锁定"某校某专业"，不是专业组
- 无组内调剂问题，不存在"被调剂到不喜欢的专业"
- 志愿数量通常更多（80-96个）
- 核实重点：专业代码、选科要求、招生计划数、历年该专业录取位次

**操作要点：**
```
1. 按位次筛选院校范围（逻辑同院校专业组模式）
2. 核实每个专业志愿：专业代码+选科要求+计划人数+历年专业录取位次
3. 排序原则：同梯次内最想读的专业排前面
4. 服从调剂含义：在该院校内调剂到其他未满专业（范围比专业组模式更宽）
5. 不存在"组内专业混入"问题，但要注意同校不同专业的位次差异
```

---

### 传统志愿/顺序志愿流程要点

适用情形：部分尚未完成新高考改革的省份，或某些特殊批次

**⚠️ 严禁直接套用平行志愿的冲稳保逻辑**

顺序志愿的核心差异：
```
第一志愿权重极大
第二志愿通常只能录取第一志愿未满额的院校
"冲第一志愿"失败后，后续志愿选择极为有限
```

**操作要点：**
```
1. 第一志愿必须慎重，通常选择录取概率70%以上的院校
2. 第二志愿选择往年录取率很低（第一志愿生源充足）的院校作保底
3. 不能把第一志愿设为"冲档"，否则落榜风险极高
4. 具体规则以当年省教育考试院公告为准
```

## 核心原则（院校专业组模式）

**用位次，不用分数**

分数受当年题目难度影响，跨年不可比。位次反映考生在全省同科目类别中的相对竞争力，是唯一可跨年比较的基准。

```
✅ 正确：考生位次12000 vs 某校2025年录取最低位次12353
❌ 错误：考生分数641 vs 某校2025年录取最低分642
```

**物理类和历史类完全分开比较**

新高考省份物理类和历史类是两个独立的招生体系：
- 物理类位次只和物理类院校专业组比较
- 历史类位次只和历史类院校专业组比较
- 同一所学校对两类分别设置专业组，录取完全独立

## 主流程（院校专业组模式，Step 1-7）

### Step 1：确定位次基准

- 以考生**省内同科目类别位次**为唯一比较基准
- 确认位次对应的分数段密度（了解每1分对应多少位次）
- 严格区分：物理类位次 ≠ 历史类位次

### Step 2：确定梯次分界

```
冲档：目标院校录取位次 < 考生位次（更难进），差距建议<3000位
稳档：目标院校录取位次 > 考生位次，差距100-5000位
保底：目标院校录取位次 > 考生位次，差距5000位以上
```

具体分界可根据考生风险偏好调整，但逻辑不变。

### Step 3：构建目标院校池

**按位次匹配院校层次（物理类/历史类通用）：**

| 位次区间（参考） | 主要院校层次 |
|----------------|------------|
| 全省前1% | 顶尖985/C9 |
| 前1-3% | 985/强势211 |
| 前3-8% | 211/双一流 |
| 前8-20% | 省属强校/双一流 |
| 前20%以后 | 省属本科/高职强校 |

详细穷举指南见 `references/exhaustion-guide.md`

### Step 4：专业组核实（最关键步骤）

**必须用官方文件核实，不能只依赖AI查询**

核实清单：
```
□ 组号是否正确（AI容易给错组号）
□ 组内专业构成（必须逐专业确认）
□ 选科要求（物理+化学 vs 物理+不限 vs 历史+不限等）
□ 招生类别（物理类组 vs 历史类组，不要搞混）
□ 办学地点（本部/分校区/异地校区）
□ 是否中外合作/国家专项/联合培养（通常排除）
□ 招生计划人数（人数极少=波动风险大）
□ 学费是否异常（较高收费组）
□ 组内有无不可接受专业混入
```

**数据来源优先级：**
```
1️⃣ 各省《招生考试之友》/《招生专业目录》（最高权威）
2️⃣ 各省教育考试院官网投档数据
3️⃣ 各校官网当年招生章程
4️⃣ 阳光高考平台（gaokao.chsi.com.cn）
5️⃣ AI辅助查询（需核实，仅供参考）
```

详细核实方法见 `references/data-sources.md`

**用户提供官方 PDF 时：**

- 读取 `references/ocr-workflow.md`
- 优先运行 `scripts/extract_admission_pdf.py` 做目标院校/组号的定向提取
- 若 PDF 无文字层，再按需使用 `--ocr`
- OCR 结果只标注为“需复核”，不能直接当最终事实
- 输出时保留页码、关键片段和置信度，方便用户回看官方原文

### Step 5：服从调剂判断

**核心逻辑：**
```
组内所有专业均可接受 → ✅ 服从调剂
组内有绝对不接受的专业 → 移除该志愿（不是不服从）
```

**为什么不选"不服从调剂"：**
投档进某专业组后，若6个专业志愿均未录上且不服从调剂，直接**退档**。
退档 = 本批次所有后续志愿全部作废，不继续检索。代价极大。

**判断"可接受"的方法：**
- 列出组内所有专业
- 逐一问考生：如果被调剂到这个专业，能接受4年吗？
- 有任何一个明确不接受的 → 移除该志愿
- 全部接受 → 服从调剂

### Step 6：志愿排序

**同梯次内排序原则：**
```
优先级1：院校层次（985 > 211 > 双一流 > 省属）
优先级2：考生意愿（最想去的排前面）
优先级3：深造竞争力（有读研/读博意向时参考保研率）
优先级4：位次（同层次同意愿时，位次小的排前）
```

**组内专业志愿排序（1-6位）：**
- 多数院校"分数优先"：最想读的专业放第1位
- 少数院校"志愿优先"或有"专业级差"：填报前确认各院校专业分配规则

### Step 7：最终核验

```
□ 本技能分析结果已标注为参考意见，未当作官方结论直接提交
□ 志愿数量是否合理（质量优于数量，不为填满而填满）
□ 院校代码+专业组代码逐一核对（最容易手误）
□ 所有服从调剂已确认
□ 专业志愿1-6顺序已排好
□ 填报时间窗口确认
□ 提交后截图保存
```

## 常见坑（必读）

详见 `references/common-pitfalls.md`

最高频的5个错误：
1. **AI给错组号**：许多院校AI给的组号和实际不符，必须用官方文件核实
2. **组号跨年失效**：部分院校每年重新编号，2025年组号≠2026年组号
3. **专业位次≠组线**：某专业录取位次远高于该组整体最低位次
4. **学校整体线≠专业组线**：同一学校不同专业组位次可能差距万位
5. **不服从调剂≠安全**：不服从直接退档，本批次所有志愿全部作废
